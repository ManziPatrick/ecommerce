## üöÄ Quick Deploy Steps

### 1. Backend (Render) - 5 minutes

```bash
# 1. Push to GitHub
git add .
git commit -m "Ready for deployment"
git push origin main

# 2. Go to render.com ‚Üí New ‚Üí Blueprint
# 3. Connect GitHub repo
# 4. Render auto-detects render.yaml
# 5. Click "Apply"
```

**Update these environment variables in Render dashboard:**
```bash
ALLOWED_ORIGINS=https://your-app.netlify.app
CLIENT_URL_PROD=https://your-app.netlify.app
```

**Your backend URL:** `https://your-service-name.onrender.com`

### 2. Frontend (Netlify) - 3 minutes

```bash
# Option 1: Netlify CLI
npm install -g netlify-cli
cd client
netlify init

# Option 2: Netlify Dashboard
# Go to netlify.com ‚Üí New site ‚Üí Import from Git
```

**Set environment variable in Netlify dashboard:**
```bash
NEXT_PUBLIC_API_URL=https://your-backend.onrender.com
```

**Your frontend URL:** `https://your-app.netlify.app`

### 3. Update Backend CORS

After getting your Netlify URL, update in Render:
```bash
ALLOWED_ORIGINS=https://your-actual-app.netlify.app
```

## üîë Required Environment Variables

### Backend (Render)
```bash
# Auto-generated by Render
SESSION_SECRET=<auto>
COOKIE_SECRET=<auto>
ACCESS_TOKEN_SECRET=<auto>
REFRESH_TOKEN_SECRET=<auto>

COOKIE_DOMAIN=.onrender.com
```

### Frontend (Netlify)
```bash
NEXT_PUBLIC_API_URL=https://your-backend.onrender.com
NODE_ENV=production
```

## üêõ Common Issues

### CORS Error
- Update `ALLOWED_ORIGINS` in Render with your Netlify URL
- Redeploy backend

### Session Not Persisting
- Check `SESSION_SECRET` and `COOKIE_SECRET` are set
- Verify `COOKIE_DOMAIN=.onrender.com`

### Build Fails
- Check logs in Render/Netlify dashboard
- Verify `package.json` has all dependencies
